# 08 — Internationalisation (i18n)

## Objectif

Mettre en place un système d'internationalisation complet pour le dashboard. Actuellement les textes sont hardcodés en français — on veut supporter plusieurs langues (FR, EN, DE, etc.) avec détection automatique de la langue du navigateur, changement dynamique, et persistance du choix.

## Architecture

Approche inspirée de Tunet : fichiers JSON de traduction + module utilitaire + context React. Pas de dépendance externe (pas de react-i18next ou similar) — on reste léger.

```
src/i18n/
├── index.ts          // Setup, détection, export du hook
├── locales/
│   ├── fr.json       // Français (langue par défaut)
│   ├── en.json       // Anglais
│   ├── de.json       // Allemand (optionnel)
│   └── es.json       // Espagnol (optionnel)
└── types.ts          // Types des clés de traduction
```

## Étape 2 : Module i18n

### `src/i18n/types.ts`

```typescript
import type fr from './locales/fr.json';

/** Type des clés de traduction (dérivé du fichier FR de référence) */
export type TranslationKeys = typeof fr;

/** Langues supportées */
export type SupportedLanguage = 'fr' | 'en' | 'de' | 'es';

export const SUPPORTED_LANGUAGES: { code: SupportedLanguage; label: string }[] = [
  { code: 'fr', label: 'Français' },
  { code: 'en', label: 'English' },
  // Ajouter d'autres langues ici
];

export const DEFAULT_LANGUAGE: SupportedLanguage = 'fr';
```

### `src/i18n/index.ts`

```typescript
import { createContext, useContext, useState, useCallback, type ReactNode } from 'react';
import fr from './locales/fr.json';
import en from './locales/en.json';
import type { SupportedLanguage, TranslationKeys } from './types';
import { DEFAULT_LANGUAGE, SUPPORTED_LANGUAGES } from './types';

export { SUPPORTED_LANGUAGES, DEFAULT_LANGUAGE } from './types';
export type { SupportedLanguage } from './types';

// ── Chargement des traductions ────────────────────────────────────────────────
const translations: Record<SupportedLanguage, TranslationKeys> = {
  fr,
  en: en as unknown as TranslationKeys,
};

// ── Détection automatique de la langue ────────────────────────────────────────
function detectLanguage(): SupportedLanguage {
  // 1. Vérifier le localStorage
  const stored = localStorage.getItem('ha-dashboard-lang');
  if (stored && SUPPORTED_LANGUAGES.some(l => l.code === stored)) {
    return stored as SupportedLanguage;
  }

  // 2. Vérifier la langue du navigateur
  const browserLang = navigator.language.split('-')[0].toLowerCase();
  if (SUPPORTED_LANGUAGES.some(l => l.code === browserLang)) {
    return browserLang as SupportedLanguage;
  }

  // 3. Fallback
  return DEFAULT_LANGUAGE;
}

// ── Résolution de clé imbriquée ───────────────────────────────────────────────
// t('widgets.weather.conditions.sunny') → "Ensoleillé"
function resolve(obj: Record<string, unknown>, path: string): string {
  const keys = path.split('.');
  let current: unknown = obj;
  for (const key of keys) {
    if (current == null || typeof current !== 'object') return path;
    current = (current as Record<string, unknown>)[key];
  }
  return typeof current === 'string' ? current : path;
}

// ── Context ───────────────────────────────────────────────────────────────────
interface I18nContextValue {
  language: SupportedLanguage;
  setLanguage: (lang: SupportedLanguage) => void;
  t: (key: string, params?: Record<string, string | number>) => string;
}

const I18nContext = createContext<I18nContextValue | null>(null);

export function I18nProvider({ children }: { children: ReactNode }) {
  const [language, setLanguageState] = useState<SupportedLanguage>(detectLanguage);

  const setLanguage = useCallback((lang: SupportedLanguage) => {
    setLanguageState(lang);
    localStorage.setItem('ha-dashboard-lang', lang);
  }, []);

  const t = useCallback((key: string, params?: Record<string, string | number>): string => {
    let text = resolve(translations[language] as unknown as Record<string, unknown>, key);
    
    // Fallback sur le français si la clé n'est pas trouvée
    if (text === key && language !== DEFAULT_LANGUAGE) {
      text = resolve(translations[DEFAULT_LANGUAGE] as unknown as Record<string, unknown>, key);
    }

    // Remplacement des paramètres : t('msg', { name: 'John' }) → "Hello John"
    if (params) {
      for (const [k, v] of Object.entries(params)) {
        text = text.replace(`{${k}}`, String(v));
      }
    }

    return text;
  }, [language]);

  return (
    <I18nContext.Provider value={{ language, setLanguage, t }}>
      {children}
    </I18nContext.Provider>
  );
}

export function useI18n() {
  const ctx = useContext(I18nContext);
  if (!ctx) throw new Error('useI18n must be used within I18nProvider');
  return ctx;
}
```

## Étape 4 : Utiliser dans les widgets

Exemple dans WeatherCard :

```typescript
import { useI18n } from '@/i18n';

export function WeatherCard() {
  const { t } = useI18n();
  
  // Avant : const label = CONDITION_FR[weather.state];
  // Après :
  const label = t(`widgets.weather.conditions.${weather.state}`);
  
  // Avant : const DAYS_FR = ['dim', 'lun', ...];
  // Après :
  const days = t('widgets.weather.days'); // retourne le tableau
  // Ou bien accéder par index dans le JSON
}
```

## Étape 5 : Language Switcher

Ajouter un sélecteur de langue dans les paramètres du dashboard (ou dans le EditButton) :

```typescript
import { useI18n, SUPPORTED_LANGUAGES } from '@/i18n';

function LanguageSelector() {
  const { language, setLanguage } = useI18n();
  
  return (
    <select
      value={language}
      onChange={(e) => setLanguage(e.target.value as SupportedLanguage)}
      className="bg-white/5 text-white text-sm rounded-lg px-3 py-2 border border-white/10"
    >
      {SUPPORTED_LANGUAGES.map(lang => (
        <option key={lang.code} value={lang.code}>
          {lang.label}
        </option>
      ))}
    </select>
  );
}
```

## Vérification

- [ ] La langue est détectée automatiquement au premier chargement
- [ ] Le choix de langue est persisté dans localStorage
- [ ] `t('widgets.weather.conditions.sunny')` retourne "Ensoleillé" en FR, "Sunny" en EN
- [ ] Les clés manquantes tombent en fallback sur le français
- [ ] Le language switcher change la langue de tout le dashboard instantanément
- [ ] Les paramètres dynamiques fonctionnent : `t('msg', { name: 'John' })`

## Migration progressive

Tu n'as pas besoin de migrer tous les textes d'un coup. Continue d'utiliser les textes hardcodés dans les widgets existants, et migre-les progressivement vers le système i18n quand tu les modifies. Le plus important est que l'infrastructure soit en place.
